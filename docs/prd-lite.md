# PRD Lite

## Product

Property OS Template is a public-safe operating workspace and agent control plane for owners, agencies, and implementers who want to launch an owner-approved property intelligence system.

## Outcomes

- Owner can add approved property facts.
- Renter can self-serve common information.
- Agent drafts stay owner-reviewed.
- Listing drafts are channel-ready but not auto-published.
- Implementer can fork, validate, install, and sell a managed service.
- Owner can assign one measurable mission to one specialist and inspect the resulting proof.
- No approval token can authorize a different tenant, actor, scope, action, or expired proposal.

## Required Surfaces

- property profile
- approved knowledge base
- listing draft studio
- inquiry draft and approval
- maintenance triage
- vacancy pipeline
- weekly owner dashboard
- agent run audit
- MCP dry-run server
- Vercel portal wiring
- hosted runtime checklist
- partner implementation guide
- typed MCP resources, prompts, tools, auth, and health endpoints
- specialist team manifest and bounded swarm topology
- proposal, decision, receipt, apply, verify, and undo control contract

## Release Criteria

- `npm run validate` passes
- privacy scan passes
- MCP smoke passes
- MCP adversarial and authenticated HTTP smoke tests pass
- public examples are sample-safe
- owner approval rules are visible
- partner can explain excluded scope
- every consequential action remains blocked or receipt-controlled
- provider/model selection is release-pinned configuration, not a hardcoded product claim

# Contributing

Contributions should keep the template safe, operational, and public-friendly.

## Local Checks

Run before proposing changes:

```bash
npm run validate
```

## Rules

- Use public-safe sample data only.
- Keep AI as drafting, triage, planning, and review support.
- Do not add automatic renter-facing commitments.
- Do not add scraper-based posting.
- Do not store secrets or `.env` values.
- Keep implementation docs practical enough for a small owner or agency to use.

## Useful Contribution Areas

- more validation cases
- country-specific compliance checklists
- database adapter examples
- owner-dashboard metrics
- safe integration runbooks
- better setup automation
